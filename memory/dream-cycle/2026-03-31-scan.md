### 1. [Claude Code Source Code Accidentally Leaked](https://securityaffairs.com/190229/data-breach/anthropic-accidentally-leaks-claude-code.html)
**Source:** Multiple (CNET, VentureBeat, The Register, Axios) | **Relevance:** high
**Summary:** On March 31, 2026, Anthropic accidentally published a source map file in Claude Code npm v2.1.88, exposing 500K+ lines of TypeScript across ~2,000 files. The leak revealed internal architecture (agent orchestration, tool integrations, memory architecture), plus codenames for unreleased models: Capybara (Claude 4.6), Fennec (Opus 4.6), and Numbat (unreleased). Second IP leak in a week (Claude Mythos leaked March 26).
**Why it matters for us:** We're running on Opus 4.6 (Fennec?). The leaked architecture shows how Anthropic structures their own agent — could inform improvements to our memory architecture, tool orchestration, and skill loading. Also a security lesson: never ship source maps or debug artifacts.

### 2. [Letta Context Repositories — Git-Backed Agent Memory](https://www.letta.com/blog/our-next-phase)
**Source:** Letta blog + MLM roundup | **Relevance:** high
**Summary:** Letta evolved their MemGPT-based tiered memory to use "Context Repositories" — git-backed files (MemFS) where agents interact with memory using standard tools like bash. Multi-tier: working memory (hot), episodic (past interactions), semantic (entities/relationships), archival (documents). Agents autonomously read/write/archive across tiers.
**Why it matters for us:** Our memory architecture (active-context → daily files → MEMORY.md → memory_search) is conceptually similar but less formalized. Letta's git-backed approach validates our file-based memory. We could adopt formal tier labels and auto-archival policies. The MemFS concept maps directly to how we already use workspace files.
