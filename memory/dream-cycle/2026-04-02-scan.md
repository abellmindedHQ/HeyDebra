### 1. [Claude Code Source Code Leak](https://www.latent.space/p/ainews-the-claude-code-source-leak)

**Summary:** Anthropic accidentally leaked ~500K lines of Claude Code source via a debugging source map in npm package v2.1.88. ### 2. [State of AI Agent Memory 2026 — Mem0 Benchmark Report](https://mem0.ai/blog/state-of-ai-agent-memory-2026)
**Source:** Mem0 Blog (published Apr 2) | **Relevance:** high
**Summary:** Comprehensive benchmark of 10 memory approaches using the LOCOMO dataset. Full-context is most accurate (72.9%) but 17s p95 latency. Mem0g (graph-enhanced) hits 68.4% with 2.59s p95 and 90% fewer tokens. Mem0's ecosystem now has 21 framework integrations including LangChain, LlamaIndex, CrewAI, and Google ADK.
**Why it matters for us:** Our memory system is file-based (MEMORY.md + daily notes + memory_search). The benchmark shows graph-enhanced memory (Mem0g) closes to within 5% of full-context accuracy at 91% less latency. Worth evaluating whether our SQLite-backed hybrid search could benefit from a graph-enhanced retrieval layer, especially as SecondBrain grows.

### 3. [1-Bit Bonsai 8B — First Commercially Viable 1-Bit LLM](https://prismml.com/news/bonsai-8b)
**Source:** PrismML / HN Show | **Relevance:** medium
**Summary:** PrismML launched Bonsai 8B: a true 1-bit model (all layers) with 8.2B params in 1.15GB. 14x smaller than 16-bit equivalent, runs on iPhone at 44 tok/s. Average benchmark 70.5 vs Llama's 67.1. Apache 2.0 license. Requires forked llama.cpp.
**Why it matters for us:** Local LLM on Mac mini M4 for lightweight tasks (classification, triage, quick answers) without API costs. Could run alongside our main Claude stack for non-critical work. At 1.15GB it's trivial to host.
