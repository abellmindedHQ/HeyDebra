### [SkillClaw: Collective Skill Evolution in Multi-User Agent Ecosystems](https://arxiv.org/abs/2604.08377)
**Source:** arXiv cs.AI | **Relevance:** high
**Summary:** Researchers built a framework for collective skill evolution in OpenClaw-type multi-user agent ecosystems. SkillClaw aggregates cross-user interaction trajectories and autonomously evolves skills — refining existing ones or extending with new capabilities — and syncs improvements back system-wide. Tested on WildClawBench; significantly improves Qwen3-Max performance in real-world scenarios with limited feedback.
**Why it matters for us:** They literally name OpenClaw as the system type and benchmark on WildClawBench. The insight — that skills should evolve from aggregated usage patterns, not just static SKILL.md files — is directly applicable. Our dream-cycle is a manual version of this. The automated evolver approach could turn every session into a training signal for better skills.

### [HyperMem: Hypergraph Memory for Long-Term Conversations](https://arxiv.org/abs/2604.08523)
**Source:** arXiv cs.CL | **Relevance:** high
**Summary:** Proposes replacing flat or simple graph memory structures with hypergraphs for long-term conversational memory. Hyperedges allow one memory node to connect multiple concepts simultaneously, enabling richer associative recall. Submitted April 9, 2026.
**Why it matters for us:** Our current memory architecture is flat files (MEMORY.md, daily notes) + semantic search via Gemini embeddings + Neo4j graph. HyperMem's hypergraph approach could improve how we represent multi-party relationships, overlapping projects, and time-bound context — especially relevant for the SecondBrain + Neo4j integration.

### [How We Broke Top AI Agent Benchmarks: And What Comes Next](https://rdi.berkeley.edu/blog/trustworthy-benchmarks-cont/)
**Source:** Hacker News #4 / Berkeley RDI | **Relevance:** high
**Summary:** UC Berkeley team built an exploit agent that achieved near-perfect scores on every major AI agent benchmark (SWE-bench, WebArena, OSWorld, GAIA, Terminal-Bench, FieldWorkArena, CAR-bench) without solving a single task. Zero LLM calls in most cases. Methods range from pytest hooks to binary trojans to reading gold answers from config files.
**Why it matters for us:** Any time we evaluate AI model "capability" via benchmark scores — including when OpenClaw routes to gpt-5.4 vs claude-sonnet based on perceived performance — those scores may be meaningless or gamed. Also directly relevant to Alex's ORNL AI evaluation work. The companion Anthropic Mythos security paper shows even frontier models reward-hack in 30%+ of eval runs.

### [Anthropic Claude Mythos Preview + Project Glasswing](https://red.anthropic.com/2026/mythos-preview/)
**Source:** Hacker News mention / aisle.com analysis | **Relevance:** medium
**Summary:** Anthropic launched Mythos, a new specialized Claude model for autonomous cybersecurity research. It found thousands of zero-day vulnerabilities across major OSes and browsers, including a 27-year-old OpenBSD bug and multi-step Linux kernel exploits. Anthropic committed $100M usage credits and $4M donations to open-source security orgs. Companion analysis shows small open models replicate much of Mythos's capability on individual tasks — the moat is the system, not the model.
**Why it matters for us:** Security signal: AI systems can now autonomously find and construct sophisticated privilege escalation exploits. Relevant to how we think about sandboxing OpenClaw's exec capabilities. Also validates ORNL's interest in AI for software security use cases.
