# night-swimming-email

Scan Gmail inbox, classify every email, archive the noise, and generate a GTD-style triage report saved to SecondBrain.

**⚠️ Use a capable model** — classification requires judgment. Recommended: `anthropic/claude-sonnet-4-6` or better.

---

## Inputs

- **Account** (default: `alexander.o.abell@gmail.com`): Gmail account to process
- **Report date** (default: today): YYYY-MM-DD for the output filename
- **Dry run** (optional flag): classify but don't archive anything — just report

## Classification Rules

### Archive (remove from inbox silently)

| Category | Examples |
|----------|---------|
| Receipts & order confirmations | Amazon, Shopify, PayPal, Stripe, Uber, DoorDash |
| Shipping notifications | "Your order has shipped", FedEx/UPS/USPS tracking |
| Marketing & newsletters | Unsubscribe links present, mass-sent, no direct ask |
| Social notifications | LinkedIn likes/views, GitHub stars, Twitter follows |
| Calendar noise | "Event updated", duplicate calendar invites with no change |
| Bank/credit alerts (routine) | Balance alerts, statement ready (NOT fraud alerts) |
| FYI forwarded content | No action requested, no deadline |
| Job board digests | LinkedIn Jobs, Indeed, etc. |

### Keep in Inbox (do NOT archive)

| Category | Examples |
|----------|---------|
| 🔴 URGENT | Any subject with "urgent", "action required", payment failures, account suspended |
| 🟠 ACTION NEEDED | Direct asks, replies needing response, approval requests, deadlines |
| 🟡 FYI (personal) | From real humans Alex knows, even if no immediate action |
| 💳 Payment failures | "Your payment failed", "subscription cancelled", "card declined" |
| 📅 Deadlines | Tax docs, renewal notices, filing deadlines |
| 🔒 Security | 2FA codes (recent), security alerts, breach notifications |
| 📩 Direct from contacts | Anyone in Google Contacts with a real relationship |

**When in doubt → KEEP.** Archiving is reversible; missing something critical is not.

## Workflow

### Step 1 — Fetch Inbox

```bash
# List all inbox messages (unread first, then all)
gog gmail list --account alexander.o.abell@gmail.com --label INBOX --limit 200 --format json

# If > 200, paginate with --page-token from previous result
```

Capture: message ID, from, subject, date, snippet, labels, read status.

### Step 2 — Classify Each Email

For each message:
1. Check sender against known-noise patterns (see Classification Rules above)
2. Check subject line for action signals
3. Check snippet for urgency/deadline signals
4. Assign category: `archive` | `urgent` | `action` | `fyi` | `personal`
5. Note reason for classification (1 line, used in report)

**Flag immediately** (add to urgent list regardless of other signals):
- "payment failed" / "card declined" / "subscription cancelled"
- "account suspended" / "unusual activity"
- Subject contains "deadline" + a date
- From IRS, ORNL HR, legal domains

### Step 3 — Archive Noise

```bash
# Archive by removing INBOX label (moves to All Mail, not deleted)
gog gmail archive --account alexander.o.abell@gmail.com --message-id <ID>
```

Archive in batches. Log each archived message ID + reason.

Skip archiving if:
- Dry run mode is enabled
- Any uncertainty about classification
- Message is < 24 hours old and from a real person

### Step 4 — Generate Report

Save to `/Users/debra/SecondBrain/Reflections/Daily/YYYY-MM-DD-email-triage.md`:

```markdown
---
date: YYYY-MM-DD
type: email-triage
account: alexander.o.abell@gmail.com
---

# 📬 Email Triage — YYYY-MM-DD

**Processed:** [N] emails | **Archived:** [N] | **Kept:** [N] | **Flagged:** [N]

---

## 🔴 Urgent / Needs Immediate Attention

- **[Sender]** — [Subject] _(why flagged)_
- ...

## 🟠 Action Items

- **[Sender]** — [Subject] _(what's needed)_
- ...

## 🟡 FYI / Personal

- **[Sender]** — [Subject] _(brief note)_
- ...

## 💳 Payment / Subscription Flags

- **[Sender]** — [Subject] _(status)_
- ...

---

## 🗑️ Archived ([N] messages)

<details>
<summary>Show archived list</summary>

| Sender | Subject | Reason |
|--------|---------|--------|
| [sender] | [subject] | [receipt / newsletter / notification / etc] |

</details>

---

*Generated by night-swimming-email skill*
```

### Step 5 — Summary Report to User

After saving the report, return a concise summary:

```
📬 Email triage complete — YYYY-MM-DD
- Processed: N emails
- Archived: N (receipts, newsletters, notifications)
- Kept: N
- 🔴 Urgent: [list subjects]

- Report: SecondBrain/Reflections/Daily/YYYY-MM-DD-email-triage.md
```

## Edge Cases

- **Inbox empty**: Report "Inbox clear ✅" and exit
- **gog auth error**: Stop, report auth failure, do not proceed
- **> 500 messages**: Process in batches of 100, warn Alex if inbox is unusually large
- **Reply thread from real contact**: Always keep, even if the latest message looks like noise

---

## Dry Run Mode

If invoked with "dry run" or "just classify, don't archive":
- Run all classification steps
- Skip Step 3 (no archiving)
- Report includes "⚠️ DRY RUN — nothing archived" banner
- Useful for tuning classification rules before first real run
