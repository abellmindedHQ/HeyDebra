### 1. [Anthropic Post-Mortem: Claude Code Quality Issues Were Harness Bugs](https://www.anthropic.com/engineering/april-23-postmortem)
**Source:** Anthropic Blog (Apr 23) | **Relevance:** high
**Summary:** Anthropic published a detailed post-mortem confirming that weeks of reported Claude Code degradation were caused by three harness-level bugs, not model changes: (1) reasoning effort silently downgraded from high to medium on Mar 4, (2) caching bug clearing thinking history every turn from Mar 26, (3) system prompt capping responses at 25/100 words from Apr 16. All fixed by Apr 20 in v2.1.116+. Usage limits reset for all subscribers.
**Why it matters for us:** We stayed on Opus 4.6 partly because of Opus 4.7 vulnerability reports (52% vuln rate per Forbes). But if quality issues were harness bugs not model bugs, Opus 4.7 may be safer than we thought. Re-evaluation warranted. Also confirms our Claude Code git reset bug (TOOLS.md) was real — update our version guidance to v2.1.116+.

### 7. [HN: Verification Is Now The Bottleneck, Not Generation](https://www.developersdigest.tech/blog/what-hacker-news-gets-right-about-ai-coding-agents-2026)
**Source:** Hacker News / Dev Digest | **Relevance:** high
**Summary:** Consensus forming that AI coding workflow maturity ladder is: model → workflow → skills → orchestration → verification. The bottleneck has shifted from "can AI write code?" to "can we verify AI output fast enough?" Red-team skills where agents argue with each other catching more bugs than single-model review.
**Why it matters for us:** Directly relevant to our "report-without-verifying" pattern (6+ corrections). We need a verification step in our workflow — not just "screenshot it" but an automated adversarial check. Also validates our GAN thinking approach (SOUL.md Generator/Discriminator mode).
