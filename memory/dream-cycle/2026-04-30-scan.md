### 1. [Claude Code Refuses Requests Mentioning "OpenClaw"](https://twitter.com/theo/status/2049645973350363168)
**Source:** HN #4 (1001 points, 563 comments) | **Relevance:** high
**Summary:** Reports that Claude Code refuses or charges extra when commits or prompts mention "OpenClaw." Follows Anthropic's April 4 policy discontinuing Claude Pro/Max subscriptions with third-party agent frameworks. Users must now use pay-as-you-go API keys, potentially 50x cost increase.
**Why it matters for us:** We run on Claude Opus via API keys, so the subscription ban doesn't affect us directly. But the commit-scanning behavior is concerning. If our codebase references "OpenClaw" (it does everywhere), we need to verify our Claude Code sessions aren't being throttled or refused. This also signals growing tension between Anthropic and the OC ecosystem.

### 2. [Anthropic Managed Agents with Persistent Memory (Public Beta)](https://platform.claude.com/docs/en/managed-agents/memory)
**Source:** Anthropic docs, SD Times | **Relevance:** high
**Summary:** Managed Agents now have filesystem-based persistent memory with up to 8 memory stores per session, concurrent access, audit trails, versioning, and rollback. Memory stores are mounted as directories agents can read/write using file tools. Beta header required: `managed-agents-2026-04-01`.
**Why it matters for us:** This is basically what we built manually with MEMORY.md + memory/*.md + active-context.md. Anthropic is now offering it as a first-class feature. Worth watching for architecture inspiration, but our current system gives us more control and works model-agnostically.

### 3. [PyTorch Lightning Supply Chain Attack (Shai-Hulud Malware)](https://semgrep.dev/blog/2026/malicious-dependency-in-pytorch-lightning-used-for-ai-training/)
**Source:** HN #15 (340 pts), Semgrep, The Hacker News | **Relevance:** high
**Summary:** Malicious versions 2.6.2/2.6.3 of the `lightning` package uploaded to PyPI on Apr 30. Steals SSH keys, .env files, Git/cloud credentials, crypto wallets. Attempts to inject worm payload into 50 Git branches. PyPI quarantined the project. Extension of "Mini Shai-Hulud" campaign that hit npm earlier.
**Why it matters for us:** We use pip packages and npm extensively. This is the same supply chain attack pattern flagged in DC #23 (SAP/axios/Bitwarden campaigns). Our Mac mini doesn't run PyTorch Lightning, but the escalation from npm to PyPI to ML packages is alarming. Need to audit our installed packages.

### 4. [IBM Granite 4.1: 8B Model Matching 32B MoE](https://huggingface.co/ibm-granite/granite-4.1-8b)
**Source:** HN #16 (282 pts), HuggingFace | **Relevance:** medium
**Summary:** IBM releases Granite 4.1 8B under Apache 2.0. Dense decoder-only, 131K context, 12 languages, trained on 15T tokens. Strong at tool calling, RAG, code gen, summarization. Available on Ollama.
**Why it matters for us:** A high-quality 8B model that can run locally on our Mac mini (M2). Could serve as a local fallback for when Gemini API goes down (our recurring single-point-of-failure issue). Tool calling support makes it viable for agent tasks. Worth testing via Ollama.

### 7. [Honker: SQLite-based Durable Queues + Cron](https://honker.dev/)
**Source:** HN #20 (186 pts) | **Relevance:** medium
**Summary:** New Rust/SQLite extension providing durable queues, event streams, pub/sub, and cron scheduler all in a single SQLite file. Node.js bindings available. Atomic enqueue with business writes.
**Why it matters for us:** QMD already uses SQLite for search indexing. Honker could complement it for task queuing. Our current cron system works, but if we ever need durable task queues (e.g., for Night Swimming batch jobs or capture-agent backlog), this is a clean zero-dependency option.

### 8. [Mozilla Opposes Chrome's Prompt API](https://github.com/mozilla/standards-positions/issues/1213)
**Source:** HN #7 (602 pts) | **Relevance:** medium
