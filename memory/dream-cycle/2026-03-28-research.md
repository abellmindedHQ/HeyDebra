## Deep Dive 1: jai — Lightweight Agent Sandboxing

### What It Is
jai is a Linux-only (currently) sandbox tool from Stanford's Secure Computer Systems lab. It provides three isolation modes:
- **Casual:** Copy-on-write overlay on home dir, CWD writable, rest read-only. Runs as your user. Protects integrity but NOT confidentiality (process can still READ your SSH keys, API tokens, etc).
- **Strict:** Runs as unprivileged `jai` user with empty private home. Full UID-based confidentiality. Doesn't work with NFS.
- **Bare:** Your UID but empty private home. Medium confidentiality.

Key limitations: No network isolation. Casual mode doesn't protect secrets. Granted directories (CWD) have full read/write. Not a replacement for Docker/VMs for real security.

### How It Applies to Us
We run on macOS (Darwin arm64), so jai is **NOT directly usable** right now — it's Linux-only using kernel namespaces and overlayfs. However:
- If we ever move to a Linux VPS for OpenClaw, jai becomes immediately relevant.
- The CONCEPT applies: our coding-agent skill spawns sub-agents with `bypassPermissions` and full filesystem access. Even on Mac, we could adopt the principle of narrowing the CWD grant.
- Our Mac mini has 1Password CLI, SSH keys, personal data — a rogue coding agent could read all of it.
- macOS sandbox-exec (deprecated) or Apple's App Sandbox could partially replicate this, but neither is as ergonomic.

### Implementation Path
1. **Immediate (trivial):** Add a note to coding-agent SKILL.md about sandboxing best practices. Ensure sub-agents are spawned with explicit `cwd` pointing to project directories, never home.
2. **Short-term (moderate):** Research macOS containment options. sandbox-exec profiles, or Docker Desktop for coding agent runs.
3. **Long-term (significant):** If OpenClaw adds a Linux deploy target (VPS, Docker), integrate jai as default wrapper for coding agent spawns.

### Trade-offs
- **Gain:** Reduced blast radius from rogue coding agents. Peace of mind.
- **Cost:** macOS isn't supported. Docker adds startup overhead. May break tools that need home dir access.
- **Verdict:** Not actionable today on Mac, but the principle of CWD-scoping is immediately applicable.

### Recommendation
**Watch it.** Not usable on macOS today. Adopt the CWD-scoping principle in coding-agent skill immediately.

## Deep Dive 2: Superpowers — Agentic Coding Workflow

### What It Is
A comprehensive coding workflow framework by Jesse Vincent (obra). It's a collection of "skills" (markdown instruction files) that automatically activate based on context. The key innovation is the **subagent-driven-development** pattern:

1. **Brainstorming:** Agent asks clarifying questions, produces a spec in digestible chunks
2. **Planning:** Breaks spec into 2-5 minute tasks with exact file paths and verification steps
3. **Subagent dispatch:** Fresh agent per task (no context pollution from previous tasks)
4. **Two-stage review:** After each task, a separate spec-reviewer agent checks compliance, then a code-quality-reviewer checks quality
5. **TDD enforcement:** RED-GREEN-REFACTOR. Literally deletes code written before tests.
6. **Final review:** When all tasks done, whole implementation reviewed before merge

The key insight: "An enthusiastic junior engineer with poor taste, no judgement, no project context, and an aversion to testing" — that's what coding agents ARE. Plan accordingly.

### How It Applies to Us
Our current coding-agent skill is basic: spawn agent with task, hope for the best. Superpowers highlights what we're missing:

1. **No spec phase.** We go straight from request to code. No structured brainstorming or validation.
2. **No two-stage review.** Our agents self-review but nobody independently checks their work.
3. **No TDD enforcement.** Tests are optional, not mandated.
4. **Context pollution.** We sometimes reuse agent sessions instead of dispatching fresh per task.
5. **No plan decomposition.** Large tasks aren't broken into verified chunks.

### Implementation Path
1. **Immediate (moderate):** Update coding-agent SKILL.md to include a spec validation step before coding begins. Add "fresh agent per task" as a best practice.
2. **Short-term (moderate):** Create a "code-review" skill that can be invoked after coding agent completes. Separate reviewer sub-agent.
3. **Medium-term (significant):** Port the full Superpowers workflow into OpenClaw skill format — brainstorm → plan → dispatch → review → merge pipeline.
4. **Alternative (trivial):** Just install Superpowers as a Claude Code plugin and reference it in CLAUDE.md for coding projects.

### Trade-offs
- **Gain:** Much higher code quality. Fewer bugs, better architecture, less rework. Autonomous multi-hour coding sessions become viable.
- **Cost:** More token usage per task (3 agents instead of 1). Slower for simple fixes. Overhead for tiny changes.
- **Verdict:** The two-stage review pattern alone is worth adopting. The full workflow is overkill for our typical tasks (skills, config, one-off scripts) but essential for real product code (HoldPlease, Mirror).

### Recommendation
**Do it now (partial).** Install Superpowers plugin for Claude Code immediately. Adopt the two-stage review pattern in our coding-agent skill. Full workflow port is medium-term.

## Deep Dive 3: MemMA — Memory Cycle Coordination

### What It Is
MemMA (Memory-augmented Multi-Agent) is a research framework that treats the memory lifecycle as a coordinated multi-agent process rather than isolated read/write operations. Key components:

**Forward path (construction + retrieval):**
- **Meta-Thinker:** Analyzes conversations and produces structured guidance about what's important, redundant, or conflicting. Steers both memory writes and query formation.
- **Memory Manager:** Executes atomic operations (ADD, UPDATE, DELETE, NONE) based on Meta-Thinker guidance.
- **Query Reasoner:** Iterative retrieval with a "Refine-and-Probe" loop. Instead of one-shot search, it diagnoses what evidence is missing and refines queries over multiple turns.

**Backward path (repair):**
- **Probe Generation:** After each session, synthesizes QA pairs testing factual recall, cross-session reasoning, and temporal inference.
- **In-situ Verification:** Retrieves from memory and attempts to answer the probes.
- **Evidence-Grounded Repair:** Failed probes trigger SKIP/MERGE/INSERT operations to fix the memory bank.

### How It Applies to Us
Our memory architecture maps roughly to MemMA's concepts but without the coordination:

| MemMA Component | Our Equivalent | Gap |
|---|---|---|
| Meta-Thinker | Manual judgment in daily notes | No strategic guidance on what to store |
| Memory Manager | Direct file writes to memory/*.md | No atomic operations, no dedup |
| Query Reasoner | memory_search (single-shot) | No iterative refinement |
| Probe Generation | None | No automated quality testing |
| Self-Repair | Manual MEMORY.md updates during heartbeats | No automated repair |

The biggest gap is the **backward path**. We never systematically verify our memory is correct. We could have contradictions, stale info, or missing facts and never know until we hit a failure.

### Implementation Path
1. **Immediate (moderate):** Add a "memory probe" step to the dream cycle. Generate 5-10 QA pairs from recent daily notes, test them against memory_search, flag discrepancies.
2. **Short-term (moderate):** Build an automated memory dedup check. Scan MEMORY.md + recent dailies for contradictions or stale info.
3. **Medium-term (significant):** Implement iterative memory_search — when first search doesn't find what's needed, refine query and try again (could be a wrapper function or skill).
4. **Long-term (major):** Full Meta-Thinker pattern — before writing to memory, analyze what's already there and decide ADD/UPDATE/DELETE.

### Trade-offs
- **Gain:** Higher memory accuracy. Fewer "I should have known that" moments. Contradictions caught early. Stale info pruned.
- **Cost:** Additional token usage per session for memory maintenance. Complexity in memory management. Risk of over-pruning if repair logic is wrong.
- **Verdict:** The probe-and-repair concept is immediately valuable and low-risk. The full multi-agent coordination is research-grade and overkill for now.

### Recommendation
**Do it now (memory probe step).** Add a memory verification phase to dream cycle. Queue the rest for later.
