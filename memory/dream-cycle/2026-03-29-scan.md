### 2. [Claude Code `git reset --hard` Bug (GitHub #40710)](https://github.com/anthropics/claude-code/issues/40710)
**Source:** HN (#9, 194 points) + GitHub | **Relevance:** HIGH
**Summary:** Claude Code silently runs `git fetch origin && git reset --hard origin/main` every 10 minutes via programmatic git operations (libgit2, not spawning git binary). Destroys all uncommitted changes to tracked files. Confirmed across 4 sessions, 95+ reflog entries. Worktree repos are immune.
**Why it matters for us:** We use Claude Code as a coding agent spawned via OpenClaw. If any of our agent sessions trigger this behavior, it could wipe uncommitted work on the HeyDebra repo or project repos. Need to verify our coding-agent skill isn't affected, and consider using git worktrees as a safety measure.
