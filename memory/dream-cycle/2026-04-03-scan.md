### 1. [Claude Agent SDK: Claude Code as a Library](https://medium.com/@shivanshmay2019/claude-agent-sdk-deep-dive-what-it-means-to-use-claude-code-as-a-library-773aea121787)
**Source:** Anthropic / Medium deep dive | **Relevance:** high
**Summary:** Anthropic renamed the Claude Code SDK to "Claude Agent SDK" and released it as a full library (Python + TypeScript). Ships with 9 built-in tools (Read, Write, Edit, Bash, Glob, Grep, WebSearch, WebFetch, AskUser), 18 lifecycle hook events, custom tools as in-process MCP servers, subagent spawning with isolated contexts, and session persistence via session_id.
**Why it matters for us:** This is basically what OpenClaw does but as a library. The hook system (PreToolUse, PostToolUse, etc.) could solve our process-narration problem with a hard gate on external sends. The subagent architecture mirrors our sessions_spawn. Worth evaluating as a complementary tool or inspiration for OpenClaw skill improvements.

### 2. [Claude Desktop Control + Dispatch](https://the-decoder.com/claude-code-and-cowork-now-let-anthropics-ai-take-control-of-your-mac-or-windows-desktop/)
**Source:** Anthropic (Apr 3, 2026) | **Relevance:** high
**Summary:** Claude can now directly control Mac and Windows desktops (open apps, navigate browsers, populate spreadsheets) via Claude Cowork and Claude Code. "Dispatch" enables remote computer control from anywhere. Available for Pro/Max users.
**Why it matters for us:** This overlaps with our browser automation and could complement VisionClaw. If Dispatch can work alongside OpenClaw, we could control Alex's Mac mini remotely from his phone. Worth testing when Alex is back.

### 3. [Google TurboQuant: 6x Memory Compression for LLMs + Vector Search](https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/)
**Source:** Google Research / ICLR 2026 | **Relevance:** high
**Summary:** Two-stage, training-free, model-agnostic compression pipeline that reduces KV cache memory by 6x and speeds up processing 8x with zero accuracy loss. Also compresses vector search indices with state-of-the-art recall. Community `turboquant` Python package available as HuggingFace KV cache drop-in.
**Why it matters for us:** Direct impact on our QMD semantic search (46MB index, 9,634 vectors). Could dramatically reduce memory footprint and speed up searches. Also relevant for any local LLM work (Bonsai 1-bit from last cycle + TurboQuant = much cheaper local inference).

**Source:** Anthropic Research | **Relevance:** medium
