## Second Brain Pipeline v3

Alex spent the morning designing a full "second brain" architecture with Claude chat. Started from a simple Otter.ai → notes pipeline and evolved it into an eight-stream capture system:

1. Fireflies Desktop (virtual meetings, API backbone)
2. Fireflies Mobile (in-person, phone calls)
3. Granola Desktop (human-anchored notes from local cache)
4. TwinMind Mobile (ambient/unscheduled capture)
5. The Sentinel (custom always-on daemon on Mac mini with Whisper + VAD)
6. Chrome MCP (browser context)
7. Calendar (metadata)
8. The Debrief (active human enrichment)

Key components:
- Stream fusion engine merges multiple captures of the same event
- Neo4j graph database for relationships (people, topics, projects, action items)
- Obsidian vault for human-readable linked notes
- MCP servers connecting everything to AI assistant
- Two-node architecture: Mac mini (home base) + iPhone (mobile node)
- ~$69/mo all-in cost

Build order is phased over ~13+ weeks, starting with simplest loop (Fireflies + Granola → Obsidian) and building up.

Full architecture saved to: projects/second-brain/ARCHITECTURE.md

Switched from Otter.ai (original idea) to Fireflies.ai as the primary capture tool due to better API/webhook support.
