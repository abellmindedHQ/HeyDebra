### 3. [Karpathy's LLM Wiki — Markdown Knowledge Base Architecture](https://venturebeat.com/data/karpathy-shares-llm-knowledge-base-architecture-that-bypasses-rag-with-an)
**Source:** VentureBeat / Multiple | **Relevance:** high
**Summary:** Andrej Karpathy proposed "LLM Wiki" — LLMs maintain evolving libraries of interlinked Markdown files as a knowledge base, bypassing traditional RAG/vector DB complexity. Three layers: raw sources (immutable), wiki (LLM-maintained MD), schema (config). Obsidian recommended as front-end.
**Why it matters for us:** This is EXACTLY what we're building with SecondBrain + Obsidian + QMD. Validates our architecture. Key difference: Karpathy uses LLM as the "compiler" for the wiki. We could adopt the "incremental compilation" pattern — LLM processes raw inputs and updates wiki pages automatically instead of manual note-taking.
