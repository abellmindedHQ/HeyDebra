## BB Attachment Bug — Root Cause Found (14:05)

**Status:** Opus debugging in progress. Found smoking gun.

**What I discovered:**
1. Messages from BB are parsing (my fallback patch works)
2. BUT `message.attachments` array is EMPTY (0 attachments detected)
3. This means BB's webhook payload has attachments in a different location than OpenClaw expects
4. Example log: `baseUrl=http://localhost:1234 hasPassword=true attachmentCount=0`

**The real issue:** Not a parse failure — it's an **attachment extraction failure**. BB sends attachments somewhere, but OpenClaw's `extractAttachments(message)` function looks for `message["attachments"]` and finds nothing.

**Next step:** Added detailed logging to dump the actual message object keys to see where BB puts attachments. Once we see the structure, we can patch the extraction function to handle BB's format.

**Why Haiku failed:** Couldn't understand why attachments weren't being extracted. Opus is now comparing BB's actual payload structure to what OpenClaw expects.
