## Deep Dive: Claude Opus 4.7 — Upgrade Assessment

### What It Is
Anthropic's latest production model, GA as of April 16. Key improvements over Opus 4.6:
- **Self-verification:** Model checks its own outputs before reporting back. This is a behavioral change, not just accuracy improvement. It "devises ways to verify its own outputs."
- **Better vision:** Higher-resolution image processing. Useful for screenshot verification, document analysis.
- **Efficiency gain:** "Low-effort Opus 4.7 ≈ medium-effort Opus 4.6." We could get equivalent quality at lower token cost, or better quality at same cost.
- **Instruction following:** More precise, "more opinionated" — won't just agree with the user.
- **Cyber safeguards:** From Project Glasswing. Auto-detects and blocks prohibited cybersecurity uses. Legitimate security use requires Cyber Verification Program.
- **Pricing:** Unchanged from Opus 4.6 ($5/$25 per M tokens).

Third-party validation:
- 13% resolution improvement on 93-task coding benchmark (over 4.6)
- Solves 4 tasks neither 4.6 nor Sonnet 4.6 could
- Better at reporting missing data instead of hallucinating
- Faster median latency

### How It Applies to Us
1. **Self-verification directly addresses our #2 correction pattern** (report-without-verifying, 6x occurrences). A model that naturally verifies outputs before reporting could reduce or eliminate this failure mode.
2. **Efficiency gain means cost savings.** If low-effort 4.7 ≈ medium-effort 4.6, we're getting more per dollar.
3. **Better vision** helps with screenshot verification workflows we've needed.
4. **"More opinionated"** aligns with SOUL.md directive for GAN thinking — don't hedge, say what you think.
5. **Deprecation context:** Opus 4.0 retires June 15. We're on 4.6, which is safe, but moving to 4.7 keeps us on the latest.

### Implementation Path
1. Update gateway config: change `anthropic/claude-opus-4-6` to `anthropic/claude-opus-4-7`
2. Test for 24h (one full dream cycle + GSD + heartbeat cycle)
3. Verify no regressions in BB message handling, cron outputs, tool calling
4. If clean, make permanent

**Difficulty:** Trivial (config string change)
**Risk:** Low (same pricing, backward-compatible API)

### Trade-offs
- **Gain:** Better self-verification, efficiency, vision, instruction following
- **Cost:** Potential behavioral changes we don't anticipate. The "more opinionated" personality could interact with SOUL.md's sassiness in unexpected ways.
- **Mitigation:** 24h test period before committing

### Recommendation
**Do it now.** This is a straightforward, low-risk upgrade with direct benefits to our known correction patterns. Propose to Alex in next interaction.

## Deep Dive: Anthropic "Project Deal" — Agent-to-Agent Commerce

### What It Is
Internal Anthropic experiment (December 2025, published April 27, 2026):
- 69 employees, ~$100 budget each, Craigslist-like marketplace in Slack
- AI agents (Claude) listed items, negotiated, and completed deals autonomously
- 500+ items listed, 186 transactions, $4K+ total value
- Real goods exchanged afterward (board games, keyboards, snowboard, artwork)
- Four parallel versions tested: Opus 4.5 vs lighter models
- Key finding: More powerful models → better negotiation outcomes
- Participants would pay for this as a service

### How It Applies to Us
1. **HeyDebra as commerce agent.** Alex's vision of "every person gets their own AI companion" includes handling real-world tasks. Project Deal proves agent-to-agent negotiation works today. HeyDebra could eventually handle:
   - Scheduling negotiations (meeting times, venue bookings)
   - Price comparison and procurement
   - Simple contract negotiation
   - Real estate search and comparison (Tyler Fogarty connection)

2. **Paperclip implications.** The Abellminded "fully automated company" could use agent-to-agent patterns for vendor negotiations, partnership discussions, supply chain management for the merch store.

3. **Model capability matters.** Opus 4.5 outperformed lighter models in negotiations. This validates running Opus for core tasks rather than cost-cutting with Haiku/Sonnet for everything.

4. **Pools/Pooli connection.** Interest-based communities + commerce agents = marketplace where community members' AI agents facilitate transactions. P.O.W.E.R. framework meets Project Deal.

### Implementation Path
This is research/vision, not immediate implementation. Steps:
1. Track Project Deal follow-ups from Anthropic (they said this was just a pilot)
2. When building Pools MVP, incorporate agent-to-agent commerce as a future feature
3. Consider "Debra as procurement agent" skill for personal shopping/comparison

**Difficulty:** Significant (long-term product direction, not a quick skill)
**Risk:** N/A (watching, not building)

### Trade-offs
- **Gain:** Vision alignment, product direction clarity
- **Cost:** No immediate cost — this is a "watch" item
- **Concern:** Trust. Would Alex trust an AI agent to negotiate on his behalf? Given his financial discipline focus (bankruptcy at 26, credit card guardrails), autonomous spending agents need very strict controls.

### Recommendation

## Deep Dive: Home Assistant 2026.4 — Upgrade Gap Analysis

### What It Is
HA 2026.4 release ("Infrared never left the chat") includes:
- **Infrared as first-class citizen:** IR remotes become smart controllers for TVs, ACs, etc.
- **Purpose-specific automation triggers:** Think in "real-world concepts" (someone arrives home, motion detected) instead of raw entity states
- **Matter lock management:** Full lock lifecycle management via Matter protocol
- **AI agent debugging:** "What is an AI-powered Assist thinking?" — transparency into AI decision-making in HA
- **Dashboard improvements:** Background colors for sections, favorites, gauge redesign, auto-height cards
- **Patch releases:** 2026.4.1 through 2026.4.4 (latest April 24)

We're on 2025.11.3. That's 5 major releases behind (2025.12, 2026.1, 2026.2, 2026.3, 2026.4).

### How It Applies to Us
1. **IR support:** Would let us control the LG projector, Samsung TV, and any IR-equipped appliance through HA. Currently these are controlled separately.
2. **AI agent debugging:** Could help fix the broken Google Home/Gemini integration (Gemini replaced Google Assistant, "Ask Home Assistant" is broken).
3. **Purpose-specific triggers:** Would simplify our automation logic. Instead of "entity state changed to on", we'd use "someone came home" which combines motion + presence + time.
4. **Matter locks:** Relevant if/when Alex adds smart locks to the apartment.
5. **5 releases behind = significant risk.** Each release may have breaking changes. Updating from 2025.11 to 2026.4 in one jump could be rough.

### Implementation Path
1. **Backup current HA config** (snapshot to /Users/debra/ha-backup/)
2. **Review breaking changes** for each intermediate release (2025.12 through 2026.4)
3. **Stage update through intermediate versions** if breaking changes are significant, or try direct jump
4. **Test integrations:** Hue (49 lights), Nest speakers, ElevenLabs TTS, ESPHome
5. **Verify ElevenLabs TTS still works** via Cast (our custom voice)

**Difficulty:** Moderate (multi-version jump, potential breaking changes)
**Risk:** Medium (could break existing automations, TTS, light controls)

### Trade-offs
- **Gain:** IR control, AI debugging, modern dashboard, security patches
- **Cost:** Downtime risk during update, potential breaking changes, time investment
- **Mitigation:** Full backup before starting. Can roll back if needed (HA Green supports snapshots).

### Recommendation
