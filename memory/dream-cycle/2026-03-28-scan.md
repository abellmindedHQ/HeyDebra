### 1. [jai — Easy Containment for AI Agents](https://jai.scs.stanford.edu/)
**Source:** HN (#7, 583 points) | Stanford SCS | **Relevance:** HIGH
**Summary:** Lightweight sandbox for running AI coding agents without giving them full machine access. One command (`jai codex`, `jai claude`), no Docker, no Dockerfiles. CWD stays writable, home dir gets copy-on-write overlay, rest is read-only. Three modes: casual, strict, bare.
**Why it matters for us:** We run Claude Code, Codex, and sub-agents with broad filesystem access on the Mac mini. jai could be a trivial safety layer, especially for coding-agent skill spawns. Alex's machine has personal data, 1Password, SSH keys — blast radius reduction is real.

### 2. [Superpowers — Agentic Skills Framework](https://github.com/obra/superpowers)
**Source:** GitHub Trending | **Relevance:** HIGH
**Summary:** Complete software development workflow built on composable "skills" for coding agents. Enforces brainstorming → spec → plan → subagent-driven-development → TDD → code review → merge. Skills trigger automatically. Works with Claude Code, Codex, Cursor, Gemini.
**Why it matters for us:** This is conceptually similar to OpenClaw's skill system but focused purely on coding workflows. The subagent-driven-development pattern (dispatching fresh agents per task with two-stage review) could improve our coding-agent skill. The TDD enforcement and automatic code review are features we lack.

### 3. [MemMA — Multi-Agent Memory Coordination](https://github.com/ventr1c/memma)
**Source:** HuggingFace Daily Papers (arXiv 2603.18718) | **Relevance:** HIGH
**Summary:** Multi-agent framework for coordinating the full memory cycle in LLM agents. Introduces Meta-Thinker (strategic guidance), Memory Manager (atomic edits), Query Reasoner (iterative retrieval), and self-evolving memory repair via probe QA pairs. Forward path guides construction/retrieval; backward path repairs memory after each session.
**Why it matters for us:** Our memory architecture (MEMORY.md + daily files + active-context + memory_search) is manually coordinated. MemMA's self-repair concept — synthesizing QA pairs to test memory accuracy, then auto-repairing — could inspire an automated memory quality check. The "probe generation → verification → repair" loop is exactly what a dream cycle memory maintenance phase could do.

### 4. [Anatomy of the .claude/ Folder](https://blog.dailydoseofds.com/p/anatomy-of-the-claude-folder)
**Source:** HN (#6, 594 points) | **Relevance:** MEDIUM
**Summary:** Deep walkthrough of Claude Code's `.claude/` directory structure: CLAUDE.md (system prompt), CLAUDE.local.md (personal overrides), rules/ folder (path-scoped configs), custom commands, permission policies, and auto-memory. Two separate dirs: project-level (team, committed to git) and global ~/.claude/ (personal, machine-local).
**Why it matters for us:** OpenClaw uses AGENTS.md/SOUL.md/TOOLS.md which serves a similar purpose. But the `.claude/rules/` pattern — splitting instructions by concern with path-scoped frontmatter — is a cleaner architecture we could learn from. Also relevant for the coding-agent skill since we spawn Claude Code sub-agents.
