### BlueBubbles Attachment Bug — CRITICAL
**Status:** 2026-04-08 09:30 EDT  
**Impact:** I cannot see images/files you send via iMessage until I manually request them

**Root cause:** OpenClaw's BB plugin receives webhook notifications of messages but doesn't automatically fetch and surface attached media in the session context. Attachments exist in the BB database but aren't downloaded/included in inbound messages I see.

**Evidence:** 
- 2026-04-08 ~09:08 EDT: Alex sent dentist appointment screenshot (IMG_1581.png, 1MB)
- BB API shows message ROWID 6439 with full attachment metadata (1006214 bytes, 872x1163px, PNG)
- Webhook/session context shows 0 attachments for that message
- Manual fetch via `/api/v1/attachment/{guid}/download` retrieves file successfully

**Workaround:** When Alex sends images, I manually:
1. Query `/api/v1/chat/guid/message?with=attachment`
2. Find the attachment GUID
3. Fetch via `/api/v1/attachment/{guid}/download`

**Long-term fix needed:** BB plugin should enhance inbound message handler to:
- Detect attachments in webhook payload
- Fetch attachment binary via REST
- Include in session context as media references
