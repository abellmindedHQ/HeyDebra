### 1. [Claude Design — Prompts to Prototypes](https://www.anthropic.com/news/claude-design-anthropic-labs)
**Source:** Anthropic / TheRundown AI | **Relevance:** high
**Summary:** Anthropic launched Claude Design (~Apr 20), powered by Opus 4.7's vision model. It turns prompts, screenshots, and codebases into interactive prototypes, slide decks, and marketing materials. Can establish brand systems by analyzing existing codebases and mockups. Exports to Claude Code, Canva, PPTX, PDF, HTML.
**Why it matters for us:** Direct relevance to ABE-32 logo/brand work with Paperclip. Could accelerate the brand identity pipeline — generate mood boards, mockups, and collateral faster than the current Sable/Kit/Wren agent chain. Worth evaluating as a design tool alongside v0.

### 2. [MCP STDIO Vulnerability — RCE Affecting 7,000+ Servers](https://thehackernews.com/2026/04/anthropic-mcp-design-vulnerability.html)
**Source:** The Hacker News / OX Security | **Relevance:** high
**Summary:** Critical "by design" weakness in Anthropic's MCP SDK allows arbitrary command execution via the STDIO transport interface across Python, TypeScript, Java, and Rust. Affects 7,000+ servers, 150M+ downloads. CVEs issued for LiteLLM, LangChain, Flowise, Windsurf, and others. Anthropic has declined to fix, calling it "expected behavior."
**Why it matters for us:** We use Claude Code with `--permission-mode bypassPermissions`. While we're not running MCP servers externally, any future MCP integration is at risk. This compounds the GitHub agent prompt injection finding from last cycle. Security posture review needed.

### 3. [Cloudflare Agent Memory — Managed Persistent Memory for AI Agents](https://blog.cloudflare.com/introducing-agent-memory/)
**Source:** Cloudflare Blog | **Relevance:** high
**Summary:** Cloudflare launched Agent Memory (private beta, Apr 17) — a managed service that extracts information from agent conversations and makes it available via retrieval, without filling context windows. Addresses "context rot" problem. Opinionated API with retrieval-based architecture, temporal logic, and supersession handling.
**Why it matters for us:** Our memory architecture (MEMORY.md + daily files + corrections.md + memory_search) is essentially a hand-rolled version of this. The "context rot" problem they describe is exactly what we hit with stale memory entries. Their approach to temporal logic and supersession could inform improvements to our memory verification phase.

### 4. [GSD Framework for Claude Code — Plan/Execute/Verify Phases](https://www.mindstudio.ai/blog/gsd-framework-claude-code-plan-build-applications)
**Source:** MindStudio | **Relevance:** high
**Summary:** A structured framework for Claude Code that breaks large builds into Plan, Execute, and Verify phases with clean context per phase. Addresses "context compounding" where planning + coding + verification in one session degrades quality. Each phase has defined scope and output, preventing context window degradation.
**Why it matters for us:** Directly applicable to how we use Paperclip/coding agents. ABE-28/ABE-32 work showed context drift (Sable losing track of feedback). The Plan/Execute/Verify split could be implemented in our coding-agent skill to improve large project outcomes.

### 8. [Google DeepMind — Detection Asymmetry in AI Agents](https://thehackernews.com/2026/04/weekly-recap-vercel-hack-push-fraud.html)
**Source:** Google DeepMind / The Hacker News | **Relevance:** medium
