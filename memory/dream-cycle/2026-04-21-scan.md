### 1. [OpenAI Chronicle: Long-Term Memory for AI Agents](https://developers.openai.com/codex/memories/chronicle)
**Source:** OpenAI / Reddit / HN | **Relevance:** high
**Summary:** OpenAI launched "Chronicle," a persistent memory system for Codex/ChatGPT that captures screenshots, runs OCR, and stores memories as local Markdown files. Opt-in for Pro subscribers on macOS. Claims 40% productivity boost in early testers. Privacy concerns due to screen capture approach.
**Why it matters for us:** Direct competitor to our memory architecture (MEMORY.md + daily notes + qmd semantic search). Our approach is text-native and file-based already — we're ahead of Chronicle's design philosophy. But their "cross-session context" framing validates our dream-cycle + active-context.md pattern. Worth studying their memory pruning/dashboard UX for ideas.

### 2. [Claude Code Removed from Pro Plan (A/B Test)](https://www.wheresyoured.at/news-anthropic-removes-pro-cc/)
**Source:** Anthropic / The Register / XDA | **Relevance:** high
**Summary:** Anthropic briefly removed Claude Code from new Pro ($20/mo) signups, restricting it to Max plan. Head of Growth called it "a small test on ~2% of new prosumer signups." Reversed within days but signals infrastructure strain from rapid consumer growth. Anthropic revenue now $30B+ run rate.

Watch for API pricing changes. Also: validates Alex's bet on running his own infrastructure (OpenClaw) vs depending on consumer tiers.

### 3. [MCP STDIO Vulnerability — Systemic Supply Chain Risk](https://www.ox.security/blog/the-mother-of-all-ai-supply-chains-critical-systemic-vulnerability-at-the-core-of-the-mcp/)
**Source:** OX Security / SecurityWeek / The Register | **Relevance:** high
**Summary:** OX Security revealed MCP's STDIO interface allows unsanitized command execution — a "by design" flaw Anthropic won't fix, saying sanitization is the developer's responsibility. Affects Python, TypeScript, Java, Rust SDKs. Multiple CVEs issued for LiteLLM, LangChain, LangFlow, GPT Researcher. Four exploitation families identified including zero-click prompt injection in AI IDEs.
**Why it matters for us:** We don't use MCP servers currently, but Paperclip and future tool integrations could. This is a systemic risk in the AI agent ecosystem. Our sandboxed OpenClaw approach is inherently safer, but we should audit any MCP integrations before adopting them. Covered in last night's cycle too — still actively developing.

### 5. [Google TurboQuant — 6x KV Cache Compression](https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/)
**Source:** Google Research / InfoQ / Towards Data Science | **Relevance:** medium
**Summary:** Google's TurboQuant compresses KV caches to 3-3.5 bits per value with near-zero accuracy loss. Up to 8x speedup on H100 GPUs. Uses PolarQuant (random rotation for compression) + QJL error correction. Presenting at ICLR 2026 and AISTATS 2026.
**Why it matters for us:** Indirect but significant. If this ships in production models, longer context windows become cheaper. Our qmd semantic search and SecondBrain pipeline benefit from models that can handle more context. Watch for integration into Gemini/Claude inference.

### 6. ["Claudy Day" — Claude Chat Session Hijacking](https://thehackernews.com/2026/04/google-patches-antigravity-ide-flaw.html)
**Source:** HN / Hacker News discussions | **Relevance:** medium
**Summary:** A vulnerability codenamed "Claudy Day" can silently hijack a Claude chat session and exfiltrate data with a single click. No additional integrations or tools required. The attack vector is through constrained tools whose inputs aren't strictly validated.
**Why it matters for us:** We run Claude as our primary model. While OpenClaw's architecture provides some isolation, we should understand the attack surface. Our BB/iMessage integration means an exfiltrated session could expose personal data. Monitor Anthropic's response.

### 7. [Cloudflare Agent Memory + Sandboxes (Agents Week)](https://blog.cloudflare.com/agents-week-in-review/)
**Source:** Cloudflare Blog | **Relevance:** medium
**Summary:** Cloudflare launched Agent Memory (persistent memory service), Artifacts (Git-compatible versioned storage), Sandboxes (persistent isolated environments with shell/filesystem), and Outbound Workers for zero-trust egress proxy. Also introduced "Agent Readiness score" for websites.
**Why it matters for us:** Cloudflare's "agentic cloud" vision validates the agent infrastructure direction. Their Sandboxes concept is similar to OpenClaw's execution model. The Agent Readiness score could matter for abellminded.com SEO if AI crawlers become a traffic source.

### 8. [Graph Memory Goes Production in 2026](https://mem0.ai/blog/state-of-ai-agent-memory-2026)
**Source:** Mem0 / beam.ai / Multiple | **Relevance:** medium
**Summary:** Graph memory (vs vector memory) has moved from experimental to production. Graph memory retrieves facts connected through relationships rather than just semantic similarity. GraphRAG combining knowledge graphs with LLMs is becoming standard for enterprise AI. LOCOMO benchmark standardizes long-term conversational memory evaluation.
**Why it matters for us:** Our Neo4j + SecondBrain + qmd architecture IS graph memory. We're already doing this. But our Neo4j is offline/noisy (3,067 Person nodes, only ~115 real people). The industry is validating our approach while our implementation needs cleanup. Priority: get Neo4j healthy and GraphRAG working.

### 9. [Antigravity IDE Prompt Injection → Code Execution](https://thehackernews.com/2026/04/google-patches-antigravity-ide-flaw.html)
**Source:** HN / Hacker News | **Relevance:** medium
**Summary:** Google's agentic IDE "Antigravity" had a prompt injection flaw enabling code execution through the find_by_name tool. Attack chain: stage malicious script, trigger via seemingly legitimate search. Pattern applies to any AI agent that ingests untrusted data + has execution tools + production secrets.
**Why it matters for us:** This pattern is directly relevant to OpenClaw's skill execution model. We ingest untrusted data (emails, messages, web content) and have execution tools. The Ren (CIO) data governance framework (ABE-56) should address this.
