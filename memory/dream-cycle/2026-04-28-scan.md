### 1. [Claude Opus 4.7 GA — Self-Verification & Cyber Safeguards](https://www.anthropic.com/news/claude-opus-4-7)
**Source:** Anthropic Blog | **Relevance:** high
**Summary:** Opus 4.7 is now generally available with notable gains in advanced software engineering, better vision (higher resolution images), and a new self-verification behavior where it checks its own outputs before reporting back. Also introduces cyber safeguards from Project Glasswing — the model has reduced cybersecurity capabilities vs Mythos Preview, with automatic detection/blocking of prohibited cyber uses.
**Why it matters for us:** We're on Opus 4.6. Opus 4.7 offers "low-effort 4.7 ≈ medium-effort 4.6" efficiency, meaning we could get similar quality at lower token cost. The self-verification behavior would help with the "report-without-verifying" correction pattern. Same pricing ($5/$25 per M tokens). Direct upgrade path.

### 2. [Claude Opus 4.0 & Sonnet 4.0 Deprecation — June 15, 2026](https://www.mindstudio.ai/blog/claude-sonnet-4-opus-4-deprecation-migration-guide)
**Source:** MindStudio/Anthropic | **Relevance:** high
**Summary:** The original Claude Opus 4.0 and Sonnet 4.0 releases will be retired June 15, 2026. API calls to pinned versions (claude-opus-4-0, claude-sonnet-4-0) will fail after that date. Migration to 4.5, 4.6, or 4.7 required. Also: 1M context window beta for Sonnet 4.5/4.0 retires April 30.
**Why it matters for us:** We're on claude-opus-4-6 which is safe. But any Paperclip agents or sub-agents pinned to older model strings need audit. The 1M context beta retiring April 30 (tomorrow!) could affect any workflows using extended context on older models.

### 3. [Anthropic "Project Deal" — AI Agent-to-Agent Commerce](https://www.anthropic.com/news)
**Source:** Anthropic / Artificial Lawyer | **Relevance:** high
**Summary:** Anthropic ran an internal marketplace experiment where Claude agents negotiated and completed 186 real transactions ($4K+ total) on behalf of 69 employees. Opus 4.5 agents consistently outperformed lighter models in negotiations. Participants said they'd pay for this as a service.
**Why it matters for us:** Directly relevant to Alex's vision of "every person gets their own AI companion." If agent-to-agent commerce is viable, HeyDebra could eventually handle procurement, scheduling negotiations, and real transactions. Also validates the "more capable model = better outcomes" thesis.

### 5. [GitHub Copilot Code Review → Actions Minutes Billing (June 1)](https://github.blog/changelog/2026-04-27-github-copilot-code-review-will-start-consuming-github-actions-minutes-on-june-1-2026/)
**Source:** GitHub Blog | **Relevance:** medium
**Summary:** Starting June 1, Copilot code review will consume GitHub Actions minutes in addition to AI Credits. Private repos will be billed; public repos stay free. Part of broader move to usage-based billing replacing PRUs.
**Why it matters for us:** abellminded-platform is a private repo. If we enable Copilot code review, it'll eat Actions minutes. Need to budget for this or keep reviews manual. Also affects ORNL repos if they use Copilot.

**Summary:** HA 2026.4 adds infrared as a first-class citizen (turning old IR remotes into smart controllers), purpose-specific automation triggers, Matter lock management, dashboard favorites, gauge card redesign, and — notably — "What is an AI-powered Assist thinking?" debugging for AI agents.

### 7. [VibeVoice — Open-Source Frontier Voice AI (Microsoft Research)](https://github.com/microsoft/VibeVoice)
**Source:** HN / Microsoft Research | **Relevance:** medium
**Summary:** MIT-licensed voice AI from Microsoft Research. TTS generates up to 90 minutes of multi-speaker conversational audio with natural turn-taking. ASR transcribes up to 60 minutes in one pass with speaker ID. 50+ languages. Real-time model (0.5B) also available.
**Why it matters for us:** Could be an alternative/supplement to ElevenLabs for TTS, especially for long-form content like the Be Particular audiobook. The ASR could replace Whisper for transcription. Open-source = no per-token costs after setup. Worth benchmarking against our current ElevenLabs voice.

### 8. [Hugging Face LeRobot — Critical RCE Vulnerability (CVE-2026-25874)](https://thehackernews.com/2026/04/critical-cve-2026-25874-leaves-hugging.html)
**Source:** HN / The Hacker News | **Relevance:** medium
**Summary:** Unpatched deserialization vulnerability in Hugging Face's LeRobot robotics platform allows unauthenticated remote code execution. AI inference systems often run with elevated privileges, making this especially dangerous.
**Why it matters for us:** Not directly using LeRobot, but a reminder about supply chain security. We run multiple AI services with elevated privileges (Paperclip, OC gateway). Deserialization attacks are the exact class of vulnerability that could hit our stack.

**Source:** Neo4j Blog | **Relevance:** medium
